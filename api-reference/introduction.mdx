# API Reference

The Gigstack API v2 provides endpoints for managing clients, invoices, payments, receipts, services, teams, and more.

## Base URL

```
https://api.gigstack.io/v2
```

## Authentication

All endpoints require a Bearer token in the `Authorization` header:

```
Authorization: Bearer YOUR_API_TOKEN
```

## Pagination

List endpoints support cursor-based pagination:

| Parameter | Type   | Description |
|-----------|--------|-------------|
| `limit`   | number | Results per page (default: 25, max: 100) |
| `next`    | string | Cursor for next page (from previous response) |
| `order_by`| string | Field to sort by (e.g., `created_at`) |
| `sort`    | string | Sort direction: `asc` or `desc` |

## Filtering

Most list endpoints support date range filtering:

| Parameter     | Type   | Description |
|---------------|--------|-------------|
| `created_gte` | number | Created at or after (Unix timestamp) |
| `created_lte` | number | Created at or before (Unix timestamp) |

## Error Responses

```json
{
  "success": false,
  "message": "Validation error",
  "errors": [
    { "field": "tax_id", "message": "Invalid RFC format" }
  ]
}
```

| Status Code | Description |
|-------------|-------------|
| 400 | Bad request / validation error |
| 401 | Unauthorized — invalid or missing token |
| 404 | Resource not found |
| 500 | Internal server error |
